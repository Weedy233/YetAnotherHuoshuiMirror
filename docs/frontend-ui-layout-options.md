# React + shadcn/ui 前端布局方案

## 1. 设计基调

本镜像站建议采用 React + shadcn/ui 作为主要 UI 组合，Tailwind CSS 只用于少量间距、边框、灰度层级和响应式微调。整体视觉参考 GitHub / Vercel 一类偏冷峻、克制、信息密度较高的 SaaS 风格：

- **形状**：使用小圆角，建议 `rounded-sm` / `rounded-md`，避免大圆角、胶囊按钮和过度卡片化。
- **线条**：用细边框区分区域，主边框使用 `border-border`，重要分割线可用 `border-zinc-300/70` 或暗色模式下 `border-zinc-800`。
- **灰度层次**：背景以 `bg-background`、`bg-muted/30`、`bg-card` 拉开层级；避免大面积高饱和色。
- **动效**：只保留必要的 hover、focus、loading skeleton；拒绝公式化浮动动画、过度视差和装饰性运动。
- **字体与排版**：正文 14–16px，表格/元信息 12–13px；标题层级克制，重视可扫描性。
- **组件优先级**：优先使用 shadcn/ui 的 `Button`、`Input`、`Card`、`Table`、`Tabs`、`Badge`、`Separator`、`Sheet`、`Command`、`Skeleton`、`Pagination`。

## 2. 共同信息架构

无论采用哪套布局，都建议包含以下核心页面：

| 页面 | 桌面端重点 | 移动端重点 |
| --- | --- | --- |
| 首页 | 全局搜索、数据快照、热门教师/课程、最近评价摘要 | 搜索优先，统计信息折叠为紧凑卡片 |
| 搜索结果页 | 左侧过滤/排序，右侧教师和课程结果列表 | 顶部搜索框 + Sheet 过滤器 + 分段结果 |
| 教师详情页 | 教师概览、评分统计、课程分布、评价列表 | 概览卡片置顶，评价列表分页加载 |
| 课程详情页 | 课程概览、教师列表、课程相关评价 | 教师列表和评价用 Tabs 切换 |
| 数据说明页 | 快照时间、数据字段、免责声明、反馈入口 | 纯文档式单栏布局 |

## 3. 方案 A：GitHub Repository 风格的信息台布局

### 适用场景

适合评价数据浏览为主、用户希望快速搜索教师并扫描评价的场景。该方案信息密度最高，也最接近 GitHub 的 repository / issue list 体验。

### 桌面端布局

```text
┌──────────────────────────────────────────────────────────────┐
│ Header: Logo | Search | Teachers | Courses | About           │
├──────────────────────────────────────────────────────────────┤
│ Hero: title + snapshot badge + primary search                │
├───────────────┬──────────────────────────────────────────────┤
│ Sidebar       │ Main content                                  │
│ - 数据统计    │ - Tabs: 热门教师 / 热门课程 / 最近评价          │
│ - 筛选条件    │ - Table/List rows                              │
│ - 快照说明    │ - Pagination                                   │
└───────────────┴──────────────────────────────────────────────┘
```

### 移动端布局

- Header 压缩为 Logo + 搜索图标 + 菜单按钮。
- Sidebar 内容收进 `Sheet`，通过“筛选/说明”按钮打开。
- 主内容由表格切换成列表行，每行展示教师名、课程、总分、评价数量和一段截断评论。
- Tabs 横向滚动，保留 44px 以上触控高度。

### 组件建议

- 顶部搜索：`Command` 或 `Input + Button`。
- 列表：桌面端 `Table`，移动端使用 `Card` 风格的紧凑 list item。
- 筛选：`Select`、`Checkbox`、`Sheet`。
- 状态：`Badge` 显示数据快照、评分区间、评价数。

### 风格要点

- 页面主体最大宽度 `max-w-7xl`，左右 padding 桌面 `px-6`、移动 `px-4`。
- 卡片使用 `rounded-md border bg-card shadow-none`。
- Hover 只使用 `hover:bg-muted/50`，不做位移或阴影放大。

## 4. 方案 B：Vercel Dashboard 风格的指标面板布局

### 适用场景

适合强调“镜像站数据概况”和“评分统计”的场景，视觉更接近 SaaS dashboard。首页更像一个数据控制台，适合展示当前快照、总记录数、热门教师、评分分布。

### 桌面端布局

```text
┌──────────────────────────────────────────────────────────────┐
│ Top nav: Brand | Search | Theme | GitHub                     │
├──────────────────────────────────────────────────────────────┤
│ Page heading + snapshot/version metadata                     │
├───────────────┬───────────────┬───────────────┬──────────────┤
│ Metric card   │ Metric card   │ Metric card   │ Metric card  │
├───────────────────────────────┬──────────────────────────────┤
│ Rating distribution panel      │ Top teachers table           │
├───────────────────────────────┴──────────────────────────────┤
│ Latest / highlighted reviews                                  │
└──────────────────────────────────────────────────────────────┘
```

### 移动端布局

- 指标卡片从 4 列变为 2 列，再到 1 列。
- 评分分布和热门教师上下堆叠。
- 复杂图表改成横向条形列表，避免移动端图表缩放不可读。
- 搜索框固定在 header 下方，方便高频访问。

### 组件建议

- 指标卡：`Card` + `CardHeader` + `CardContent`，但压低高度，避免营销页式大留白。
- 数据表：`Table`，移动端改为 definition-list 式块状行。
- 评分分布：优先用纯 CSS 条形条，不引入重型图表库。
- 分区标题：`Separator` + 小号 uppercase label，塑造冷峻工具感。

### 风格要点

- 主背景 `bg-background`，指标区使用 `bg-muted/20` 做弱分区。
- 数字用 `font-mono` 或 `tabular-nums`，提升数据产品感。
- 保持统一 1px 边框，避免渐变和重阴影。

## 5. 方案 C：Docs + Search 风格的知识库布局

### 适用场景

适合把站点定位为“评价数据镜像与查询工具”，弱化社交评价氛围，强调可检索、可引用、可解释。对合规和免责声明更友好。

### 桌面端布局

```text
┌──────────────────────────────────────────────────────────────┐
│ Header: Brand | Search | Data | About | Disclaimer           │
├───────────────┬──────────────────────────────┬───────────────┤
│ Left nav      │ Main article / detail content │ Right toc     │
│ - 教师索引    │ - 教师详情                    │ - 页面锚点     │
│ - 课程索引    │ - 评论列表                    │ - 数据快照     │
│ - 数据说明    │ - 字段解释                    │ - 反馈入口     │
└───────────────┴──────────────────────────────┴───────────────┘
```

### 移动端布局

- 左侧导航进入 `Sheet`。
- 右侧目录隐藏或折叠到页面顶部。
- 教师详情和课程详情使用单栏文档式排版，评论列表按时间/评分分页。
- 顶部搜索使用 `Command` 弹层，适合快速跳转教师或课程。

### 组件建议

- 导航：`ScrollArea` + `Button` variant ghost。
- 内容：`Card` 少用，更多使用 `section + border-b`，减少卡片堆叠感。
- 页面目录：`TableOfContents` 自定义组件或普通锚点列表。
- 搜索弹层：`CommandDialog`，桌面端支持快捷键 `/` 或 `⌘K`。

### 风格要点

- 更克制的文档站风格，减少首页营销感。
- 信息以线性阅读为主，适合解释数据来源、快照时间、删除/反馈机制。
- 桌面三栏布局需要设置中间内容 `min-w-0`，避免长评论撑破布局。

## 6. 方案 D：Split View 搜索工作台布局

### 适用场景

适合高频搜索和对比多个教师/课程的用户。左侧始终展示搜索结果，右侧展示选中教师或课程详情，类似邮箱 / Linear / GitHub code search 的工作台体验。

### 桌面端布局

```text
┌──────────────────────────────────────────────────────────────┐
│ Header: Brand | global search | nav                          │
├────────────────────────┬─────────────────────────────────────┤
│ Result pane            │ Detail pane                         │
│ - search input         │ - selected teacher/course header     │
│ - filters              │ - score summary                      │
│ - result list          │ - tabs: reviews / courses / stats    │
└────────────────────────┴─────────────────────────────────────┘
```

### 移动端布局

- 默认显示搜索结果列表。
- 点选结果后进入详情页路由，而不是强行左右分栏。
- 详情页顶部提供返回搜索结果按钮，保持任务流连续。
- 筛选项放入底部或右侧 `Sheet`。

### 组件建议

- 搜索结果：`ScrollArea` + 紧凑 list item。
- 详情内容：`Tabs` 分为评价、课程、统计。
- 移动端导航：用路由状态或 URL query 保存搜索词和页码，方便返回。
- Loading：左侧 list 和右侧 detail 分别用 `Skeleton`，避免整页闪烁。

### 风格要点

- 左右面板之间用 `border-l` / `border-r`，不用阴影。
- 结果项选中态使用 `bg-muted` + 左侧 2px accent line。
- 桌面端适合 `h-[calc(100vh-56px)]` 固定工作台高度，内部滚动。

## 7. 推荐优先级

1. **首选方案 A：GitHub Repository 风格的信息台布局**。它最适合当前数据量和评价浏览场景，实现成本低，响应式清晰，容易从静态 JSON + D1 API 逐步演进。
2. **首页可吸收方案 B 的指标面板**。在首页顶部加入 3–4 个小指标卡，有助于说明镜像站数据规模和快照状态。
3. **详情页吸收方案 C 的文档站克制感**。教师详情、课程详情、数据说明页都应减少营销化卡片，突出可读性和合规说明。
4. **方案 D 可作为后续增强**。如果发现用户大量进行连续搜索和对比，再实现 split view；初版不建议优先做，移动端复杂度更高。

## 8. 响应式断点建议

| 断点 | 布局策略 |
| --- | --- |
| `<640px` | 单栏；表格改列表；筛选进入 `Sheet`；隐藏低优先级列；按钮触控高度 ≥ 40px。 |
| `640–1024px` | 两栏或紧凑 grid；保留搜索和核心 tabs；统计卡片 2 列。 |
| `>1024px` | 启用 sidebar / dashboard grid / split view；表格展示更多列；详情页增加右侧 metadata。 |

建议 CSS/Tailwind 约定：

```tsx
<main className="mx-auto w-full max-w-7xl px-4 py-4 sm:px-6 lg:px-8">
  <section className="grid gap-4 lg:grid-cols-[260px_minmax(0,1fr)]">
    {/* sidebar + content */}
  </section>
</main>
```

## 9. shadcn/ui 主题建议

建议将全站 tokens 控制在低饱和灰度体系：

```css
:root {
  --radius: 0.375rem;
}
```

组件层面：

- `Card`：`rounded-md shadow-none`。
- `Button`：默认 `rounded-md`，主按钮可使用黑/白反差，不使用大面积品牌色。
- `Input`：保留清晰边框和 focus ring，适合搜索场景。
- `Badge`：用于评分、评价数、快照状态，避免过多颜色。
- `Table`：桌面端主要数据承载；移动端不要强塞表格。

## 10. 初版页面组合建议

初版可以按以下顺序实现：

1. 首页：方案 A 主结构 + 方案 B 的小指标卡。
2. 搜索结果页：列表优先，移动端使用 Sheet 筛选。
3. 教师详情页：顶部 summary + Tabs + 评价分页。
4. 课程详情页：顶部 summary + 关联教师 + 评价分页。
5. 数据说明页：方案 C 文档式布局，包含快照时间、字段解释、免责声明和反馈方式。
